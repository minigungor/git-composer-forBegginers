Для работы с git предусмотрен репозиторий, хранилище для локальных - глобальных изменений
для этого существует команда git remote
где add - добавить репозиторий, rm - удалить репозиторий
git init --bare project - создает главный репозиторий project - в котором можно создавать другие репозитории

git remote add origin #http://github.com - origin название репозитория который будет храниться в ветке, http - адрес самого репозитория,
 может быть локальным или глобальным

для работы с ним есть команды - pull, push, fetch, clone

git clone origin repo2 - полностью копирует репозиторий, repo2 - название клона репозитория 
git push origin main - отправляет на репозиторий ветку main
git pull origin main - сравнивает локальный репозитори (наш) с главным, выводит, и сразу коммитит изменения, на ветку main
git fetch origin main - тоже самое что git pull только изменения не применяет, а выводит в отдельную ветку

git pull --rebase - возвращает обновленный репозиторий, и rebase выполняет
