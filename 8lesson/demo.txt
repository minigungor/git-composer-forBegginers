в git имеются дополнительные функции
такие как метки, stash, git merge --no-ff, git add -i

метки - git tag - функция для работы с метками, они используются для наименования 
 коммитов, для быстрого перемещенния или работы с ним

git tag - список всех тегов
git tag v1.0 - добавить метку v1.0
git tag -d v1.0 - удалить метку

git diff v1.0 v1.1 -  показать разницу между метками

обычно при push/pull теги не задействуются
для этого нужно писать флаг --tags

git stash - по сути локальное хранилище, в которое мы можем создавать стек файлов
 для будущих коммитов. Очень выручает когда нужно переключиться на файл, но 
 нужно за коммитить изменения чтобы переключиться, а так просто хранит их в хранилище

git stash - добавляет в стек
git stash list - выводит список
git stash apply - возвращает коммит из стэш
git stash drop - удаляет коммит в стэш


обычно если коммиты на ветках становятся крайними, то вместо объединения
git делает fast-forward - перемотку

если мы не хотим fast-forward - то можно слить их без него
git merge feature --no-ff

git add -i
просто интерактивная программа в консоле для добавления файлов, и просмотр их 
 изменений в индекс
