git - работает по следующей схеме
у нас имеются состояния index, и в них мы коммитим наши изменения

для создания проекта пишем
git init . #в текушей директории git init project #в аудитории project

дальше нам нужно открыть кофиг git
чтобы указать автора и почту пользователей, кто добавляет изменения

git config --global user.name = 'имя пользователя'
git config --global user.mail = 'почта пользователя'

глобально указываем для того чтобы везде были указаны данные, а не для каждого проекта в ручную

есть основные команды
git status - показывает какие файлы поменялись
git log - показывает какие изменения за все время происходили
git add . # -a - добавляет в индекс файлы, . - в текущей директории, -A все файлы которые изменились, имя_файла - только этот файл
git rm --cached # - удаляет из индекса файлы и директории которые мы укажем
git checkout - меняет положения head - благодаря которой по хэшу можно переключаться между состояниями и менять изменения
git diff - показывает изменения которые произошли - так же можно указать какие файлы или директории, и даже между состояниями
git commit - открывает файл в котором пишем текст что изменилось - git commit -m 'имя коммита' - коммитим что произошло простым текстом

имеются флажки:
-oneline - для git log - показывает только названия коммитов и дату
и тд.
