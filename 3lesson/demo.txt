Тут мы смотрели системы веток и их объединения
для создания веток пишем команду
git branch #имя ветки
git branch - все ветки выводит
git branch -M #название - меняет главную ветку на название

git merge #название ветки с которой объединяемся
бывает конфликт когда в файле в одном и том же месте происходят изменения
это называется merge conflict

как решить это - открыть файл
там будут указаны изменения из двух проблеммных веток
и чтобы их изменить вручную

git branch -d #удаляет ветку

желательно удалять ветки, на их месте просто остаются хэши

так же еще fast-forward - это когда при слиянии веток, крайними является 1 общий коммит
поэтому ветка просто по ней пробегает и происходит слияние
