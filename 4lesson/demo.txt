В гит есть помимо merge - rebase
если merge - она делает слияние, то rebase - всю ветку переносит вперед, как бы продолжая ее

если по каким нибудь причинам хотим удалить файлы из индекса
можно использовать git reset которая полностью обнулит индекс на котором находимся

есть два вида reset: --soft - когда файлы остаются с изменениями, --hard когда файлы удаляются с изменениями

так же в git есть условные указатели HEAD~2

у нас ORIG_HEAD это последний коммит на котором находится главная ветка
дальше если идти по ветке изменениям то указатель у нас будут HEAD~1, HEAD~2 и тд
то есть от самого вверха до самого низа

в git присутсвует файл
.gitignore

в нем мы пишем файлы, директории, расширения файлом - которые не будут коммититься, только будут на жестком диске
обычно в нем можно указать папки как .idea и другие от  IDE
можно даже писать условия: игнорируй те папки где есть - в них можно писать .gitkeep папки которые будут по нему игнорироваться
